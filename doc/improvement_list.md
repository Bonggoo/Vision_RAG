# 📑 Vision RAG 개선 및 고도화 요구사항 정의서

본 문서는 손글씨 노트를 바탕으로 정리된 Vision RAG 프로젝트의 개선 필요 사항 및 향후 고도화 로드맵을 정의합니다.

---

## 1. 🔒 보안과 사용자 (Security & Users)
* **로그인 과정 자연스럽게**
  * 로그인 인증 후 메인 화면으로 전환되는 과정의 UX 흐름 개선
  * 불필요한 리다이렉트나 깜빡임 현상 제거
* **구글 OAuth 계정 정리**
  * 사용하지 않거나 중복 등록된 구글 API/인증용 계정 설정 정리
* **화이트리스트 폐지**
  * 기존의 특정 사용자 접근 제한(화이트리스트)을 폐지하여 자연스러운 로그인 및 인증으로 권한 관리 방식 간소화

## 2. ⚡ 코어 기능 (Core Performance)
* **대화 품질 테스트 진행**
  * **AI 자동 평가**: LLM을 활용한 답변 품질 평가 및 벤치마크 수행
  * **수동 & 출력 검증**: 담당자(지운, 준혁)가 직접 테스트 질문을 입력하고 생성된 답변 출력을 교차 검증
* **성능 및 기능 개선**
  * **속도 개선**: RAG 파이프라인 각 단계(Phase)의 응답 속도 최적화
  * **매뉴얼 코드 반영**: 답변 내용 중에 근거가 되는 **매뉴얼 코드(참조 섹션 및 페이지 번호 등)**를 시각적으로 명시하여 답변의 신뢰성 확보
  * **대화 시 파일 업로드**: 대화 진행 중에 대화창 내부에서 직관적이고 편리하게 파일을 업로드하고 분석할 수 있는 방식 도입 검토

## 3. 📂 파일 관리 기능 (File Management)
* **문서 관리 및 편집 기능 고도화**
  * 업로드된 문서를 제조사, 모델 단위로 **삭제, 변경(메타데이터 인라인 수정), 다운로드**할 수 있는 기능 제공
* **업로드 개선 (창분리 고려)**
  * 업로드 화면 진행 상태나 결과 화면이 대화창과 독립적으로 나뉘거나(창 분리) 조화를 이루도록 설계 고려

## 4. 🎨 UX (사용자 경험 & UI)
* **앱 명칭 및 디자인 수정**
  * **'테크노트(TechNote)' 서비스명 도입 검토**: 기존 명칭을 대체할 '테크노트' 브랜딩 및 세련된 테크 감성의 다크 모드 레이아웃 적용
  * **반응형 화면 가독성 100%**: 모바일 화면에서도 텍스트가 잘리지 않고 가독성 있게 표시되도록 반응형 CSS 레이아웃 최적화

## 5. 💡 고도화 아이디어 (Future Roadmaps)
1. **다양한 문서 포맷 지원**
   * PDF 외에 직원이 작성한 오피스 파일(`.doc`, `.xlsx`, `.ppt` 등)을 추가로 분석하고 RAG 데이터로 활용할 수 있는 파이프라인 설계
2. **암묵지(개인 코멘트)의 문서화 (Memory)**
   * 단순히 대화 히스토리를 저장하는 것에서 나아가, 사용자가 남긴 개인 코멘트 및 암묵적 지식을 추출·요약하여 맞춤형 지식 베이스(Memory)로 자산화하고 축적하는 기능
3. **자주 하는 질문 (FAQ) 챗봇 방식**
   * 사용자가 자주 찾는 질문 유형을 사전 등록하여, 대기 시간 없이 즉시 최적의 답변을 매칭해주는 시나리오 결합
4. **유저 피드백 수집 및 자동 적용**
   * 생성된 답변에 대한 사용자 피드백(좋아요/싫어요 등)을 수집하여 RAG 품질 개선 학습 데이터로 자동 환류하는 파이프라인 구축
