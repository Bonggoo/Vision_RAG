## 1. 프로젝트 개요 (Overview)

본 프로젝트는 **대규모 산업용 매뉴얼(PDF)**을 효율적으로 탐색하고, 현장의 복잡한 질의에 **정확한 해답을 제공**하는 **'Vision RAG'** 시스템 구축을 목표로 합니다.

기존 벡터 DB를 **완전히 배제**하는 **Vectorless 아키텍처**와, AI가 인간처럼 문서를 탐색하는 **Agentic Search** 방식을 결합하여 **멀티모달 기반의 혁신적인 검색 환경**을 제공합니다.

---

## 2. 배경 및 목적 (Background & Objectives)

### 2.1 기존 RAG의 한계점

| 문제 | 설명 |
|------|------|
| **Vector DB의 비효율성** | 방대한 매뉴얼을 청킹(Chunking)하고 임베딩하는 데 소모되는 컴퓨팅 리소스와 DB 유지 비용이 과도함 |
| **시각적 맥락 훼손** | 기존 텍스트 분할 방식에서 도면, 표, 배선도 등의 시각적 정보가 손실됨 |
| **정보 손실** | 청킹 경계에서 문맥이 단절되어 정확한 답변 도출이 어려움 |

### 2.2 해결 방안: Agentic Vision RAG

인간 엔지니어가 매뉴얼을 찾는 방식과 동일하게 동작합니다:

1. **AI 에이전트가 목차(ToC, 계층형 인덱스)를 먼저 분석**하여 목표 페이지를 논리적으로 추론
2. **타겟 페이지만 미니 PDF로 추출**하여 원본 레이아웃 100% 보존
3. **멀티모달 LLM이 시각적 정보**(표, 도면, 알람 코드)를 **직접 분석**하고 정확한 답변 도출

> **핵심 차별점**: Vectorless + Zero Context Loss + Agentic Reasoning

## 3. 핵심 기능 요구사항 (Functional Requirements)

| **F-02** | Agentic 페이지 추론 | 사용자 질의에 대해 AI 에이전트가 목차 트리를 탐색하여 가장 연관성 높은 타겟 페이지(들)를 논리적으로 특정 | P0 |
| **F-03** | On-demand 미니 PDF 추출 | 특정된 페이지 번호를 기반으로 원본 PDF에서 타겟 페이지만 추출한 미니 PDF 생성 (PyMuPDF) | P0 |

| **F-05** | 구조화된 트러블슈팅 답변 | 분석 결과를 현장 작업자가 즉시 실행할 수 있는 스텝별 조치 가이드 형태로 포맷팅하여 제공 | P1 |

## 4. 비기능 요구사항 (Non-Functional Requirements)

| 항목 | 요구사항 |
|------|----------|
| **응답 속도** | 단일 질의 기준 End-to-End 응답 시간 30초 이내 |
| **확장성** | 동시 다수 매뉴얼(10개+) 등록 및 병렬 질의 처리 지원 |
| **정확도** | 시각 정보(표, 도면) 포함 질의에 대해 기존 텍스트 RAG 대비 정확도 향상 |
| **비용 효율** | Vector DB 인프라 비용 0원, 필요 페이지만 렌더링하여 토큰 최적화 |
| **접근성** | PC·모바일 브라우저에서 별도 설치 없이 URL 접속만으로 즉시 사용 가능 |
| **배포 용이성** | Git Push → 자동 배포(Vercel), 동료에게 URL 공유만으로 배포 완료 |

## 5. 시스템 아키텍처 및 프로세스 (Architecture)

### 5.1 Agentic 탐색 파이프라인

```
┌──────────┐    ┌─────────────────────┐    ┌──────────────┐
│  질의 입력  │───▶│  Agentic Search     │───▶│  타겟 페이지   │
│  (Query)  │    │  (목차 트리 추론)      │    │  (Page N) 도출 │
└──────────┘    └─────────────────────┘    └──────┬───────┘
                                                  │
                                                  ▼
┌──────────┐    ┌─────────────────────┐    ┌──────────────┐
│  최종 답변  │◀───│  Vision LLM         │◀───│  실시간 렌더링  │
│ (Response)│    │  (시각 정보 분석)      │    │ (Zero Loss)  │
└──────────┘    └─────────────────────┘    └──────────────┘
```

### 5.2 상세 처리 흐름

1. **질의 입력** → 사용자가 자연어로 매뉴얼 관련 질문 입력
2. **Agentic Search** → AI 에이전트가 계층형 목차 트리를 탐색하여 관련 페이지 추론
3. **타겟 페이지 도출** → 가장 연관성 높은 페이지 번호(들) 특정
4. **미니 PDF 추출** → PyMuPDF로 타겟 페이지만 추출한 미니 PDF 생성 (Zero Context Loss)
5. **Vision LLM 분석** → 멀티모달 LLM이 미니 PDF 내 시각 정보를 직접 분석
6. **최종 답변** → 구조화된 트러블슈팅 가이드 형태로 응답 생성

### 5.3 기술 스택 (Tech Stack)

| 레이어 | 기술 |
|--------|------|
| **Frontend** | Next.js 16 (React 19) + Zustand + TailwindCSS 4 |
| **Styling** | 커스텀 디자인 시스템 (글래스모피즘, oklch 컬러) |
| **Backend** | Python 3.10+ (FastAPI) |
| **PDF 처리** | PyMuPDF(fitz) 단일 의존성 (ToC 추출, 미니 PDF 추출, 썸네일 생성) |
| **AI Orchestration** | LangChain (ChatGoogleGenerativeAI) |
| **AI Model** | Gemini 2.5 Pro (Vision, PDF 네이티브 입력) |
| **Architecture** | Vectorless Agentic RAG + 2단계 추론 파이프라인 |
| **Deployment** | Vercel (Frontend) + Google Cloud Run (Backend) |
