## 1. 프로젝트 개요 (Overview)

본 프로젝트는 **대규모 산업용 매뉴얼(PDF)**을 효율적으로 탐색하고, 현장의 복잡한 질의에 **정확한 해답을 제공**하는 **'Vision RAG'** 시스템 구축을 목표로 합니다.

기존 벡터 DB를 **완전히 배제**하는 **Vectorless 아키텍처**와, AI가 인간처럼 문서를 탐색하는 **Agentic Search** 방식을 결합하여 **멀티모달 기반의 혁신적인 검색 환경**을 제공합니다.

또한, **구글 OAuth 인증 체계**와 **사용자별 멀티테넌시 격리 필터**를 통해 보안성을 강화하고, 현장에서 직접 스마트폰 카메라로 촬영한 알람 장비 이미지를 즉각 분석할 수 있는 **멀티모달 이미지 RAG**를 지원합니다.

## 2. 배경 및 목적 (Background & Objectives)

### 2.1 기존 RAG의 한계점

| 문제 | 설명 |
|------|------|
| **Vector DB의 비효율성** | 방대한 매뉴얼을 청킹(Chunking)하고 임베딩하는 데 소모되는 컴퓨팅 리소스와 DB 유지 비용이 과도함 |
| **시각적 맥락 훼손** | 기존 텍스트 분할 방식에서 도면, 표, 배선도 등의 시각적 정보가 손실됨 |
| **정보 손실** | 청킹 경계에서 문맥이 단절되어 정확한 답변 도출이 어려움 |

### 2.2 해결 방안: Agentic Vision RAG

인간 엔지니어가 매뉴얼을 찾는 방식과 동일하게 동작합니다:

1. **AI 에이전트가 목차(ToC, 계층형 인덱스)를 먼저 분석**하여 목표 페이지를 논리적으로 추론
2. **텍스트 기반 정밀 탐색**으로 섹션 내 정확한 타겟 페이지 특정
3. **타겟 페이지만 미니 PDF로 추출**하여 원본 레이아웃 100% 보존
4. **멀티모달 LLM이 시각적 정보**(표, 도면, 알람 코드)를 **직접 분석**하고 정확한 답변 도출

> **핵심 차별점**: Vectorless + Zero Context Loss + 3-Phase Hybrid Reasoning

## 5. 시스템 아키텍처 및 프로세스 (Architecture)

### 5.1 3단계 하이브리드 추론 파이프라인

```
┌──────────┐    ┌─────────────────────┐    ┌──────────────┐
│  질의 입력  │───▶│  Phase 0+1           │───▶│  섹션 특정     │
│  (Query)  │    │  문서 선택 + ToC 추론    │    │  (p.45~60)   │
└──────────┘    │  (Flash-Lite)        │    └──────┬───────┘
                └─────────────────────┘           │
                                                  ▼
┌──────────┐    ┌─────────────────────┐    ┌──────────────┐
│  최종 답변  │◀───│  Phase 3: Vision LLM │◀───│  Phase 2     │
│ (Response)│    │  미니 PDF 분석        │    │  텍스트 정밀   │
│ (Markdown)│    │  (Flash-Lite 통일)   │    │  페이지 탐색   │
└──────────┘    └─────────────────────┘    │  (Flash-Lite) │
                                           └──────────────┘
```

### 5.2 상세 처리 흐름

1. **로그인 및 인증** → 구글 계정으로 로그인하여 JWT 토큰을 발급받고, 이후 모든 API 요청 헤더에 Authorization 주입
2. **질의 입력 및 이미지 첨부** → 사용자가 자연어로 질문 입력 또는 스마트폰 카메라로 촬영한 알람 이미지(Base64) 첨부
3. **Phase 0+1** → AI 에이전트가 본인 소유의 문서 중에서 자동 선택 + 계층형 목차 트리 탐색으로 섹션 특정 (Flash-Lite 1회)
4. **Phase 2** → 특정된 섹션의 전체 텍스트 추출 후 정밀 분석으로 정확한 타겟 페이지 도출 (Flash-Lite 1회)
5. **미니 PDF 추출** → PyMuPDF로 타겟 페이지만 추출한 미니 PDF 생성 (Zero Context Loss)
6. **참조 이미지** → 타겟 페이지 썸네일(dpi=150) 생성 후 Base64로 프론트 전송
7. **Phase 3** → 멀티모달 LLM이 미니 PDF 내 시각 정보를 직접 분석 후 스트리밍 응답 (Flash-Lite 통일로 비용 및 속도 최적화)

### 5.3 기술 스택 (Tech Stack)

| 레이어 | 기술 |
|--------|------|
| **Frontend** | Next.js 15.2 (React 19) + Zustand 5 + TailwindCSS 4 |
| **Styling** | 커스텀 디자인 시스템 (글래스모피즘, oklch 컬러) |
| **State & Auth** | Zustand 5 (useAuthStore, useChatStore, useDocumentStore) |
| **Backend** | Python 3.10+ (FastAPI 0.136) |
| **PDF 처리** | PyMuPDF 1.27 (fitz) 단일 의존성 (ToC 추출, 미니 PDF 추출, 썸네일 생성) |
| **AI Orchestration** | LangChain Core 1.4 + LangChain Google GenAI 4.2 |
| **AI Model** | Gemini 3.1 Flash-Lite (추론/탐색/Vision 전 구간 통합 사용) |
| **Storage** | Google Cloud Storage (GCS) 다이렉트 업로드 및 Signed URL 다운로드 |
| **Deployment** | Vercel (Frontend) + Google Cloud Run (Backend) |
